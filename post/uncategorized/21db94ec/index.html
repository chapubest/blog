git学习笔记

提交

  • 在提交时输入多行信息 
git commit -m "第一行
第二行
第三行"
  • 修改刚刚最后一次提交的注释 
git commit --amend

更多内容可以参考https://www.jianshu.com/p/098d85a58bf1

撤销

  • 撤销当前工作区的修改
    未add到暂存区，则撤销回和版本库一样，add到暂存区后又做修改，则撤销回上一次add到暂存区后的状态 
# 撤销指定文件的修改
git checkout -- <文件名>
# 撤销所有修改
git checkout .

分支

git鼓励使用分支进行新功能的开发，而且因为git中分支通过指针实现，对分支的创建、删除、修改都十分方便快速。

  • 创建分支 
git branch <分支名称>

# 创建分支并切换到该分支  
git checkout -b <分支名称>

git switch -c <分支名称>
  • 重命名 
# 本地分支
git branch -m <旧名称> <新名称>
# 如果要同时重命名远端分支，还要执行下面的命令
git push origin <新名称>

git push -d origin <旧名称>
  • 查看分支 
# 查看本地分支
git branch

# 查看远程分支
git branch -r

# 查看所有分支
git branch -a
  • 切换分支
git checkout <分支名称>

git switch <分支名称>
  • 删除分支 
git branch -d <分支名称>
  • 合并分支
# 合并指定分支到当前分支
git merge <分支名称>

git会尝试使用Fast forward方式进行合并。
当合并过程中会出现冲突时，git通过<<<<<<<，=======，>>>>>>>标记不同分支内容，需要手动修改冲突，并重新提交(add&commit)。
git默认使用Fast forward方式合并，删除分支后，是看不到分支信息的。
可以在合并的时候禁用Fast forward模式，此时，会生成一个新的commit，就可以从历史中看到分支信息了。 

git merge --no-ff -m "commit信息" <分支名称>
  • 选择特点commit进行合并

开发中存在某次提交内容为bug修改，想要将该提交复制到其他分支上可以使用cheey-pick 

git cheey-pick <commit编号>
  • 远程操作 
# 从远程下载下所有变动，不会自动合并到本地
git fetch <远程主机名>

# 获取指定分支的变动
git fetch <远程主机名> <远程分支名>
# 从远程拉取变动，并与本地指定分支合并
git pull <远程主机名> <远程分支名>:<本地分支名>

# 如果是和当前分支合并可以简写为
git pull <远程主机名> <远程分支名>

# 也可以简写为
git pull

暂存工作区

在开发过程中需要临时处理bug，但是当前工作尚未完成(尚未commit)，可以使用stash保存工作区。

  • 保存 
git stash
  • 查看 
git stash list
  • 恢复
# 不会删除stash
git stash apply

# 恢复指定stash
git stash apply <stash@{编号}>
# 同时从stash list中删除
git stash pop
  • 删除 
git stash drop

标签

  • 查看标签 
git tag

# 查看标签信息
git show <tag名称>
  • 创建标签
    创建和删除标签都是在本地操作，如果想要同步到远程可以查看远程操作部分。 
# 默认标签打在最新提交的commit上，即HEAD
git tag <tag名称>

# 达到指定的commit上
git tag <tag名称> <commit id>

# 创建带有说明的标签
git tag -a <tag名称> -m "tag信息"
  • 删除标签
git tag -d <tag名称>
  • 远程操作 
# 推送一个标签
git push origin <tag名称>

# 一次性推送所有未推送的标签
git push origin --tags

# 从远程删除
git push origin :refs/tags/<tag名称>

日志

  • 以分支图方式查看日志 
git log --graph

gitignore文件

  • 创建gitignore文件 
touch .gitignore
